Purpose: The pre-create-then-rename intake pattern requires: (1) creating tickets with temporary slugs, (2) renaming them after the intake agent generates a descriptive slug, and (3) failing pre-created tickets if the intake agent crashes. This plan delivers the tooling foundation that the intake agent and orchestrator depend on.

<feature>
  <name>ticket rename subcommand and queued->failed transition</name>
  <files>~/.claude/skills/fixme/scripts/fixme-tools.cjs, ~/.claude/skills/fixme/references/state-machine.md</files>
  <behavior>
    ticket rename:
    - Input: ticket path + --slug flag
    - Validates slug: lowercase, hyphens, a-z0-9 only, max 60 chars, collapses repeated hyphens, trims leading/trailing hyphens
    - Empty slug after sanitization -> error
    - Missing file -> error
    - Missing --slug -> error
    - Reads ticket frontmatter, updates `slug` field and `updated` timestamp
    - Derives title from new slug (capitalize each word)
    - Updates body heading (# NUMBER: Title)
    - Computes new filename: `{number}-{new-slug}.md`
    - Writes updated content, then renames file if path changed
    - Returns JSON: { oldPath, newPath, oldSlug, newSlug, number, title }
    - If old and new paths are same (slug unchanged after sanitization): still updates frontmatter, no rename

    queued->failed transition:
    - Add 'failed' to TRANSITIONS['queued'] array
    - Requires --reason flag (consistent with all failed transitions)

    Cases:
    - ticketRename("path/0003-intake-tmp-a7b3.md", {slug: "login-button-broken"}) -> renames to 0003-login-button-broken.md, returns { oldPath, newPath, ... }
    - ticketRename("path/0003-foo.md", {slug: "Hello World!!!"}) -> sanitizes to "hello-world", renames to 0003-hello-world.md
    - ticketRename("path/0003-foo.md", {slug: "---"}) -> error: slug empty after sanitization
    - ticketRename("nonexistent.md", {slug: "x"}) -> error: file not found
    - ticketTransition("path/0003-ticket.md", "failed", {reason: "Intake failed"}) from queued -> succeeds
    - ticketTransition("path/0003-ticket.md", "failed", {}) from queued -> error: reason required
  </behavior>
  <implementation>

       - Use `fs.renameSync` for atomic rename (same filesystem guaranteed).
       - Handle the case where old heading doesn't match expected format gracefully (still update frontmatter even if heading replacement doesn't match).

    2. Add 'failed' to `TRANSITIONS['queued']` array (line 512):
