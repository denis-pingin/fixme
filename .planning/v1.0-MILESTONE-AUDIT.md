### New Findings (INT-06 through INT-10)

#### INT-06 (HIGH): max_attempts not enforced at tool level

`fixme-tools.cjs` `ticketTransition()` allows `verifying -> planning` unconditionally. The TRANSITIONS matrix does not check `current_attempt < max_attempts`. The only guard is fix-agent.md prose ("If attempts exhausted: go to Step 5"). A buggy agent or crashed re-dispatch could retry indefinitely.

- **Affects:** FIXR-05, STAT-01
- **File:** `.claude/skills/fixme/scripts/fixme-tools.cjs`, ticketTransition(), line ~733
- **Fix:** Add `current_attempt >= max_attempts` check before allowing `verifying -> planning`

#### INT-07 (MEDIUM): No implementer feedback on retry

fix-agent.md dispatches fix-implementer with static "first cycle" verifier feedback on all attempts. On retry (attempt 2+), the implementer gets no information about what the verifier found wrong. The planner dispatch has a retry variant; the implementer does not.

- **Affects:** FIXR-05
- **File:** `.claude/skills/fixme/agents/fix-agent.md`, lines 113-121
- **Fix:** Add retry variant of implementer dispatch that passes verification report path

#### INT-08 (MEDIUM): No null-guard on base_commit in SKILL.md revert

fix-agent writes `base_commit` to ticket frontmatter via Edit tool. If fix-agent crashes between `git rev-parse HEAD` and the Edit, `base_commit` remains null. SKILL.md's failure handler would run `git diff --name-only null HEAD` — invalid command.

- **Affects:** STAT-03, FIXR-04
- **File:** `.claude/skills/fixme/SKILL.md`, lines 254-262
- **Fix:** Add null-check on base_commit; fall back to `git stash` or no-op with user warning

#### INT-09 (MEDIUM): Double-write to Fix section

Sub-agents (researcher, planner, implementer) each write summary bullets to the ticket's `## Fix` section. fix-agent.md also writes the same bullets after each sub-agent returns. Result: duplicate entries in the fix section.

- **Affects:** STAT-03
- **Files:** fix-researcher.md:101-104, fix-planner.md:97-98, fix-implementer.md:71-72, fix-agent.md:53-65
- **Fix:** Remove sub-agent writes OR remove fix-agent writes (one writer per entry)

#### INT-10 (LOW): Assets path format inconsistency

investigation-agent.md example uses `assets/<ticket-number>/repro-<desc>.png` subdirectory path, but `ticketCreate` only creates a flat `assets/` directory.

- **Affects:** BROW-02
- **File:** `.claude/skills/fixme/agents/investigation-agent.md`, line 135
- **Fix:** Update example to use flat `assets/` path
