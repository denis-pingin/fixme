### New Finding

#### INT-11 (HIGH): ticketList missing files_changed field

SKILL.md commit step c.1 uses `ticket list` to retrieve `files_changed` for `git add`. But `ticketList()` returns `{ number, slug, state, title, path, dir }` — no `files_changed` field. The orchestrator has no CLI path to retrieve the changed files list without reading the raw ticket body (which violates its own Rule 2: "NEVER read ticket bodies").

- **Affects:** FIXR-01, STAT-03
- **File:** `.claude/skills/fixme/scripts/fixme-tools.cjs`, ticketList() lines 773-785
- **Fix:** Add `files_changed` to ticketList output (read from frontmatter), or add a `ticket get-field` subcommand

### Minor Items (not blocking)

1. **SKILL.md Task dispatches missing `subagent_type`** (MEDIUM) — investigation-agent and fix-agent dispatches don't specify `subagent_type: "general-purpose"`. May work if tools propagate from SKILL.md's `allowed-tools` header. Depends on runtime behavior.

2. **In-memory intake tracking not compaction-safe** (LOW) — Orchestrator's mental list of running intake agents is lost on context compaction. After compaction, auto-close may fire while a background intake Task is still running.
