Update fixme-tools.cjs to use the new ticket-centric directory layout. Every function that references the old flat `tickets/` directory must change.

**ticketCreate(sessionDir, flags):**
- Remove the `tickets/` directory dependency. Instead, scan `sessionDir` for directories matching `/^\d{4}-/` pattern that contain `ticket.md`.
- Create the ticket as `sessionDir/NNNN-slug/ticket.md` with subdirectories: `assets/`, `research/`, `plans/`, `verifications/`.
- Use `fs.mkdirSync(path, { recursive: true })` for all subdirectories.
- Return `{ path: ticketPath, dir: ticketDir, number, slug, state }` -- add `dir` field pointing to the ticket folder.
- See research Example 1 for the code pattern.

**ticketList(sessionDir, flags):**
- Scan `sessionDir` for directories matching `/^\d{4}-/` that contain `ticket.md`.
- Read `ticket.md` inside each matching directory.
- Return array with `{ number, slug, state, path, dir }` -- `path` points to `ticket.md`, `dir` points to the ticket folder.
- See research Example 2 for the code pattern.

**ticketNext(sessionDir):**
- Same scanning logic as ticketList but filtered to `state === 'queued'`.
- Return `{ path, dir, number, slug }` or `{ path: null }`.

**ticketRename(ticketPath, flags):**
- The ticket is now at `NNNN-slug/ticket.md`. Renaming means renaming the parent directory.
- Read the ticket from `ticketPath`, update frontmatter slug/title, write back to the same `ticketPath`.
- Then rename the PARENT DIRECTORY: `fs.renameSync(oldDir, newDir)` where `oldDir = path.dirname(ticketPath)` and `newDir = path.join(path.dirname(oldDir), newFolderName)`.
- Return `{ oldPath, newPath, oldDir, newDir, oldSlug, newSlug, number, title }`.

**sessionCreate(baseDir, flags):**
- Remove creation of `tickets/` and `assets/` subdirectories. The session directory just gets `session.md`. Ticket folders are created by `ticketCreate`.

**sessionList(baseDir):**
- Update ticket counting: scan for `*/ticket.md` inside session dirs instead of `tickets/*.md`.

**sessionSummary(sessionDir):**
- Update ticket scanning: scan for `*/ticket.md` inside `sessionDir` instead of `tickets/*.md`.

**Also add a new subcommand `ticket dir`:**
- `ticketDir(ticketPath)` -- given a ticket.md path, return `{ dir: path.dirname(ticketPath) }`.
- Register in the main router under `ticket` -> `dir`.

Update the existing test file (fixme-tools.test.cjs) to reflect the new directory layout. Tests that create tickets in `tickets/` subdirectory must be updated to expect `NNNN-slug/ticket.md` structure. Run the test file to verify: `node .claude/skills/fixme/scripts/fixme-tools.test.cjs`.
